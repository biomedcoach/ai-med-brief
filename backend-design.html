3 你怎么操作 —— 日常流程

每天的标准操作流程，分两步：

1

打开精选页面，挑 10 篇

晚上 9 点后，浏览器打开 http://localhost:5000，看到当天 30~40 篇候选项——每篇有标题、摘要、Quality Score 分数。你勾选 10 篇，点「确认发布」。

2

（凌晨自动）生成页面 + Git Push

publish.py 在凌晨 1:00 自动运行，生成新一天的 index.html 和文章页面，git commit + push。GitHub Pages 自动部署，几秒钟生效。

最简单的情况下：你只需要每晚打开一个网页，勾 10 篇文章，关掉。其他的全自动。

6 AI 翻译模块

translate.py 做的事情
  1. 读 candidates_YYYY-MM-DD.json
  2. 每篇调一次 SiliconFlow API（免费）
  3. 提取结果里的 JSON：en_title / en_summary / en_key_points / tags
  4. 写回 processed_YYYY-MM-DD.json
  5. 失败的文章标记 skip，不阻塞
Prompt 模板👇
System You are a professional English-language medical technology journalist. Your job is to translate Chinese medical AI news into concise, accurate English summaries that read like they came from STAT News or MedTech Dive.
User For the following Chinese article, output valid JSON with these fields:
1. en_title — an accurate, news-style English headline (max 14 words)
2. en_summary — a 120-160 word English summary covering key facts, numbers, and significance
3. en_key_points — 2-3 key bullet points (one sentence each)
4. tags — choose from: AI-diagnosis, AI-drug-discovery, AI-medical-device, Regulation, Funding, Exhibition

--- ARTICLE START ---
[中文原文标题和正文]
--- END ---

7 发布模块

凌晨 1:00 自动跑

publish.py 做的事情：

  1. 读 selected_YYYY-MM-DD.json（你勾的那 10 篇）
  2. 根据模板 HTML，替换 {{article_list}} 占位符，生成新 index.html
  3. 为每篇文章生成 articles/2026-05-26-01.md（后续文章详情页用）
  4. 更新 RSS feed
  5. Git add + commit + push
# index.html 里的文章卡片区域，用 Jinja2 模板变量 <main class="article-feed"> <div class="feed-header"> <h2>{{ date_str }} — Today's Selection</h2> <span class="feed-meta">{{ selected_count }} articles from {{ candidate_count }} candidates</span> </div> {% for article in articles %} <article class="article-card"> <div class="card-top"> <span class="card-tag tag-{{ article.tag_class }}">{{ article.tag_label }}</span> <span class="card-date">{{ article.date }}</span> </div> <h3 class="card-title">{{ article.en_title }}</h3> <p class="card-summary">{{ article.en_summary }}</p> <div class="card-points"> {% for point in article.en_key_points %} <div class="card-point">{{ point }}</div> {% endfor %} </div> </article> {% endfor %} </main>

8 你日常操作的 3 个命令

日常只需要记住这 3 个
命令 做什么

python curation_ui.py 打开精选页面 → 浏览器 http://localhost:5000 → 勾选 → 确认

9 先跑本机，再上 GitHub Actions

A

本机阶段（现在）

本地跑 Python 脚本。手动触发 crawl → translate → curation_ui → publish。不用管 GitHub，先把爬虫质量调好。

B

GitHub Actions 阶段（验证后）

把 crawl + translate 写进 .github/workflows/daily.yml，设置 cron 定时。GitHub 免费跑。精选还是你手动勾。

第一阶段（本机）就是：
python crawl.py → 看 30 篇候选质量怎么样 → 调评分权重 → 重复直到满意。
python curation_ui.py → 模拟精选流程 → 看出来效果好不好。
